## 2. 데이터 파이프라인 (Ingestion)

### PDF → 청크 → 임베딩 → 저장

```
[PDF 업로드]
    ▼
[텍스트 추출]
 ├─ 일반 PDF → pdf.js / pdfplumber
 └─ 스캔 PDF → OCR (Tesseract / Claude Vision)
    ▼
[전처리]
 ├─ 테이블 구조 보존 (물성 데이터 등)
 ├─ 메타데이터 태깅 (제품코드, 문서유형, 날짜)
 └─ 불필요 요소 제거 (머리글/바닥글, 페이지번호)
    ▼
[청킹 (Chunking)]
 ├─ 기본: 섹션 단위 분할 (제목/소제목 기준)
 ├─ 테이블: 테이블 단위로 별도 청크 (물성표 보존)
 ├─ 오버랩: 전후 2~3문장 중첩 (문맥 유지)
 └─ 청크 크기: 500~1000 토큰
    ▼
[임베딩 생성]
 └─ OpenAI text-embedding-3-small 또는 Voyage-3
    ▼
[Vector DB 저장]
 └─ 임베딩 + 원문 텍스트 + 메타데이터(제품, 문서유형, 페이지)
```

### 구조화 데이터 (DB) 연동

```
[제품 DB]  ──┐
[물성 DB]  ──┤── SQL 쿼리 기반 검색 (정확한 수치)
[규격 DB]  ──┤
[위키 문서] ──┘── 텍스트 → 임베딩 → Vector DB
```

**핵심**: 물성 수치(압축강도 52.3 MPa 등)는 **구조화 DB에서 직접 조회**하고,
서술형 내용(시공방법, 기술 해설 등)은 **RAG로 검색**하는 **하이브리드 방식**.

## 3. 기술 스택 추가사항

### 추가 기술 스택

| 구분 | 기술 | 역할 |
|------|-----|------|
| **Vector DB** | pgvector (PostgreSQL 확장) | 임베딩 벡터 저장 및 유사도 검색 |
| **Embedding** | OpenAI `text-embedding-3-small` | 텍스트 → 벡터 변환 (1536차원) |
| **LLM** | Claude API (claude-sonnet-4-6) | 답변 생성 |
| **OCR** | Tesseract.js + Claude Vision | 스캔 PDF 텍스트 추출 |
| **Reranker** | Cohere Rerank 또는 자체 구현 | 검색 결과 관련성 재정렬 |
| **Framework** | LangChain.js 또는 Vercel AI SDK | RAG 파이프라인 오케스트레이션 |
| **텍스트 추출** | pdf-parse / pdfjs-dist | PDF → 텍스트 변환 |
| **청킹** | LangChain TextSplitter | 문서 분할 |

### pgvector 선택 이유
- PostgreSQL 위에서 동작 → **별도 Vector DB 서비스 불필요**
- 기존 제품/문서 DB와 **같은 DB에서 JOIN 가능**
- 메타데이터 필터링 + 벡터 검색 동시 가능
- 소~중규모 데이터에 충분한 성능

## 4. 질문 유형별 처리 전략

| 질문 유형 | 예시 | 처리 방법 |
|----------|------|----------|
| **수치 조회** | "RM-100 압축강도?" | SQL 직접 조회 (제품 물성 DB) |
| **비교** | "RM-100과 RM-200 차이?" | SQL 조회 + LLM 비교 정리 |
| **규격 확인** | "KS F 4042 기준은?" | 규격 DB + 위키 문서 RAG |
| **시공 방법** | "보수몰탈 시공 순서?" | PDF RAG (TDS 문서) |
| **추천** | "지하주차장 방수에 적합한 제품?" | 구조화 DB 필터 + LLM 추천 |
| **시장 정보** | "글로벌 보수몰탈 시장 규모?" | 위키 문서 RAG (시장 동향) |
| **복합 질문** | "해양환경에서 KS 기준 만족하는 고강도 몰탈?" | SQL 필터 + RAG + LLM 종합 |

### 의도 분류 (Intent Detection)

```
사용자 질문 → 의도 분류기 (LLM 기반)
  ├─ LOOKUP:   정확한 수치/데이터 → SQL 쿼리
  ├─ EXPLAIN:  설명/해설 요청 → RAG 검색
  ├─ COMPARE:  비교 요청 → SQL + LLM
  ├─ RECOMMEND: 제품 추천 → SQL 필터 + LLM
  └─ GENERAL:  일반 질문 → RAG 검색
```

### UI 기능
- **출처 링크**: 답변 근거 문서 클릭 시 해당 PDF/위키 페이지로 이동
- **빠른 질문 버튼**: 자주 묻는 질문 바로가기
- **대화 이력**: 이전 질문/답변 유지
- **피드백**: 답변 유용성 평가 (👍👎) → 품질 개선 데이터
- **위키 페이지 내 인라인 질문**: 문서 열람 중 특정 부분 선택 → AI에 질문

## 6. 데이터 동기화

```
[PDF 업로드] ──→ 텍스트 추출 ──→ 청킹 ──→ 임베딩 ──→ Vector DB
                                                      ↑ 자동 동기화
[위키 문서 편집] ──→ 변경된 문서 재임베딩 ─────────────────┘
                                                      ↑ 자동 동기화
[제품 DB 수정] ──→ 관련 제품 정보 재임베딩 ────────────────┘
```

- 문서/제품 변경 시 **해당 청크만 재임베딩** (전체 재처리 불필요)
- 삭제된 문서의 벡터는 자동 제거
- 일일 배치로 전체 임베딩 정합성 검증

## 7. 안전장치

| 항목 | 대응 |
|------|------|
| **할루시네이션 방지** | 반드시 검색된 자료 기반으로만 답변, 출처 명시 필수 |
| **자료 없음 처리** | "해당 내용은 등록된 자료에서 확인되지 않습니다" 응답 |
| **수치 정확성** | 물성 데이터는 구조화 DB에서 직접 조회 (LLM 생성 금지) |
| **범위 제한** | 특수몰탈/건설 관련 질문만 답변, 범위 외 질문은 안내 |
| **버전 관리** | 답변 시 참조한 문서의 버전/날짜 표시 |
