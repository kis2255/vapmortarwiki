## 플랫폼별 UI 패턴 비교

### 1. MediaWiki (Wikipedia)

| 요소 | 구현 방식 | 우리 프로젝트 적용 |
|-----|---------|----------------|
| **네비게이션** | 좌측 고정 사이드바 + 카테고리 트리 | ✅ 채택 - 카테고리별 제품 트리 |
| **문서 레이아웃** | 본문 + 우측 요약 정보박스(Infobox) | ✅ 채택 - 제품 상세의 정보 카드 |
| **목차(TOC)** | 문서 상단 자동 생성, 접기 가능 | ✅ 채택 - 위키 문서에 TOC 적용 |
| **카테고리** | 하단 카테고리 링크, 카테고리 페이지 | ✅ 채택 - 태그 + 카테고리 |
| **검색** | 상단 검색바, 자동완성 | ✅ 채택 |
| **버전 관리** | 편집 이력, 비교(diff) | △ 간소화 적용 |
| **단점** | UI가 구식, 모바일 대응 약함 | 모던 UI로 재해석 필요 |

**핵심 차용**: Infobox(요약 정보 카드), TOC, 카테고리 시스템

---

### 2. Docusaurus (Meta)

| 요소 | 구현 방식 | 우리 프로젝트 적용 |
|-----|---------|----------------|
| **네비게이션** | 좌측 트리 사이드바 (접기/펼치기) | ✅ 채택 |
| **문서 레이아웃** | 좌측 사이드바 + 본문 + 우측 TOC | ✅ 3열 레이아웃 채택 |
| **버전 관리** | 문서 버전별 스냅샷 | △ 제품 버전에 적용 |
| **검색** | Algolia DocSearch 통합 | ✅ 전문검색 적용 |
| **모바일** | 반응형 (사이드바 → 햄버거) | ✅ 채택 |
| **Markdown** | MDX 지원 (React 컴포넌트 삽입 가능) | ✅ Markdown 에디터 |

**핵심 차용**: 3열 레이아웃, 반응형, MDX 기반 문서 관리

### 3. GitBook

| 요소 | 구현 방식 | 우리 프로젝트 적용 |
|-----|---------|----------------|
| **네비게이션** | 깔끔한 좌측 사이드바, 그룹핑 | ✅ 깔끔한 사이드바 디자인 |
| **검색** | 실시간 검색 + 미리보기 | ✅ 검색 시 미리보기 표시 |
| **첨부파일** | 문서 내 파일 임베드 | ✅ PDF 뷰어 임베드 |
| **디자인** | 미니멀, 여백 활용, 타이포그래피 | ✅ 클린 디자인 참고 |

**핵심 차용**: 미니멀 디자인, 검색 미리보기, 파일 임베드

### 4. Notion

| 요소 | 구현 방식 | 우리 프로젝트 적용 |
|-----|---------|----------------|
| **DB 뷰** | 테이블/보드/갤러리/캘린더 뷰 전환 | ✅ 제품 목록 뷰 전환 |
| **필터/정렬** | 속성 기반 필터링 | ✅ 제품 필터 (카테고리/용도/규격) |
| **페이지 트리** | 무한 중첩 페이지 | △ 2~3단계로 제한 |
| **블록 에디터** | 블록 단위 편집 | △ Markdown 에디터로 대체 |

**핵심 차용**: DB 뷰 전환(테이블/카드), 속성 기반 필터링

## 기술 제품 데이터 UI Best Practices

### 물성 데이터 표시
- **비교 테이블**: 규격 기준값 vs 실측값을 나란히 표시, 적합/부적합 시각화
- **단위 명시**: 모든 수치에 단위 표시 (MPa, %, mm 등)
- **시각적 강조**: 규격 초과 달성 시 초록색, 미달 시 빨간색

### 제품 카드 (Infobox)
```
┌─────────────────────┐
│ [제품 사진/아이콘]     │
│                     │
│ 제품코드: RM-100     │
│ 카테고리: 보수몰탈    │
│ 규격: KS F 4042     │
│ 주요강도: 52.3 MPa   │
│ 포장: 25kg/포        │
│                     │
│ [TDS] [MSDS] [인증서] │
└─────────────────────┘
```

### 검색 UX
- **통합 검색**: 제품명, 문서 제목, PDF 내 텍스트 모두 검색
- **자동완성**: 제품코드, 규격번호 자동완성
- **필터**: 검색 후 유형별 필터 (제품/문서/PDF/규격)
- **하이라이트**: 검색어 결과 내 하이라이트

## 최종 UI 설계 방향 요약

| 영역 | 채택 레퍼런스 | 구현 방식 |
|------|------------|---------|
| 전체 레이아웃 | Docusaurus | 사이드바 + 본문 + 우측 TOC (3열) |
| 사이드바 | MediaWiki + Docusaurus | 카테고리 트리, 접기/펼치기 |
| 제품 상세 | Wikipedia Infobox | 우측 정보 카드 + 본문 + 물성 테이블 |
| 위키 문서 | Wikipedia + GitBook | Markdown 기반, TOC 자동생성 |
| 제품 목록 | Notion | 테이블/카드 뷰 전환, 필터/정렬 |
| 검색 | GitBook + Algolia | 실시간 검색, 미리보기, 필터 |
| PDF 업로드 | 자체 설계 | 드래그앤드롭 + AI 자동분류 |
| 디자인 톤 | GitBook | 미니멀, 넓은 여백, 명확한 타이포 |
| 반응형 | Docusaurus | 사이드바 → 햄버거, 테이블 → 카드 |
