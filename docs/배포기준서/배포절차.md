## 3. Dockerfile

프로젝트 루트에 `Dockerfile`을 생성합니다.

```dockerfile
FROM node:20-alpine AS base

# ── 의존성 설치 ──
FROM base AS deps
WORKDIR /app
COPY package.json package-lock.json ./
COPY prisma ./prisma/
RUN npm ci
RUN npx prisma generate

# ── 빌드 ──
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# ── 런타임 ──
FROM base AS runner
WORKDIR /app
ENV NODE_ENV=production
RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs
EXPOSE 3000
ENV PORT=3000
CMD ["node", "server.js"]
```

> **참고:** standalone 빌드를 사용하려면 `next.config.ts`에 `output: "standalone"` 추가 필요.

## 7. Nginx 가상호스트 설정

```nginx
# HTTP → HTTPS 리다이렉트
server {
    listen 80;
    server_name vapm.sampyo.co.kr;
    return 301 https://vapm.sampyo.co.kr$request_uri;
}

# HTTPS
server {
    listen 443 ssl;
    server_name vapm.sampyo.co.kr;

    # 와일드카드 인증서 (*.sampyo.co.kr)
    ssl_certificate     /home/sampyopi/sam-workmanual/nginx/ssl/2026/fullchain.pem;
    ssl_certificate_key /home/sampyopi/sam-workmanual/nginx/ssl/2026/2026_sampyo.co.kr_.key.pem;

    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    # Next.js 프록시 (SSR이므로 모든 요청을 프록시)
    location / {
        proxy_pass http://127.0.0.1:8002;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 300;
        client_max_body_size 50M;
    }
}
```

### 적용 명령어
```bash
# 설정 파일 생성
sudo nano /etc/nginx/sites-available/vapm.sampyo.co.kr

# 활성화
sudo ln -sf /etc/nginx/sites-available/vapm.sampyo.co.kr \
            /etc/nginx/sites-enabled/vapm.sampyo.co.kr

# 문법 검사 후 재시작
sudo nginx -t && sudo systemctl reload nginx
```
