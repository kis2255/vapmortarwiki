# 프로젝트 대시보드 설계서

> 최종 수정일: 2026-04-07
> 참고: asanadashboard 설계 문서 (Flask 독립앱 → Next.js 통합 모듈로 재설계)

---

## 1. 개요

### 1.1 목적
삼표 PI(Performance Improvement) 프로젝트 진행 현황을 VAP 위키 내에서 **간트 차트, 테이블, 신호등 상태**로 시각화한다. Asana에서 프로젝트/태스크 데이터를 가져와 한 화면에서 조회할 수 있게 한다.

### 1.2 배경
기존 asanadashboard는 Flask 독립 앱으로 운영. 이를 VAP 위키에 **신규 메뉴**로 통합한다. 기존 "대시보드"(`/`, Home)는 그대로 유지하고, 사이드바 첫 번째 메뉴로 "프로젝트 현황"을 추가한다.

### 1.3 기존 대비 변경점

| 항목 | asanadashboard (기존) | vapmortarwiki 통합 (신규) |
|------|----------------------|--------------------------|
| 프레임워크 | Flask + 순수 HTML/JS | **Next.js 16 + React 19** |
| 스타일 | 인라인 CSS | **Tailwind CSS 4 + CSS 변수** |
| 라우팅 | `/` (단일 페이지) | **`/dashboard`** (사이드바 첫 메뉴) |
| 설정 저장 | config.json (파일) | **DB (settings 테이블) 또는 localStorage** |
| 캐싱 | 메모리 (Flask) | **Next.js API Route + 메모리 캐시** |
| 테마 | 고정 (라이트) | **라이트/다크 모드 지원** |
| 인증 | 없음 | 없음 (사내 네트워크 전용) |

## 3. API 설계

### 3.1 내부 API Routes

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/dashboard/projects` | 프로젝트 + 태스크 전체 조회 (캐시) |
| GET | `/api/dashboard/projects/[gid]/tasks` | 특정 프로젝트 태스크 조회 |
| POST | `/api/dashboard/refresh` | 캐시 강제 무효화 + 재조회 |
| GET | `/api/dashboard/config` | 설정 조회 (토큰 마스킹) |
| POST | `/api/dashboard/config` | 설정 저장 |

### 3.2 Asana API 호출

#### 토큰 검증
```
GET https://app.asana.com/api/1.0/users/me
Authorization: Bearer {token}
```

#### 프로젝트 목록
```
GET https://app.asana.com/api/1.0/projects
  ?workspace={workspace_gid}
  &archived=false
  &limit=100
  &opt_fields=name,owner,owner.name,start_on,due_on,
              current_status_update,current_status_update.color,
              num_tasks,num_incomplete_tasks,num_completed_tasks
```

#### 프로젝트별 태스크
```
GET https://app.asana.com/api/1.0/tasks
  ?project={project_gid}
  &limit=100
  &opt_fields=name,assignee,assignee.name,start_on,due_on,completed
```

### 3.3 Rate Limit 대응
- Asana API: 분당 150 요청
- 캐시 TTL: 5분 (환경변수 `ASANA_CACHE_TTL` 로 조정)
- 429 응답 시 Retry-After 헤더 대기 후 재시도

## 4. 신호등 상태 판별 로직

### 4.1 프로젝트

```typescript
function getProjectStatus(project): "green" | "blue" | "red" | "none" {
  // 1순위: Asana status_update.color 사용
  if (project.current_status_update?.color) {
    return project.current_status_update.color;
  }
  // 2순위: 자동 판별
  const today = new Date();
  const due = project.due_on ? new Date(project.due_on) : null;
  const progress = project.progress;

  if (due && due < today && progress < 100) return "red";      // 기한 초과
  if (due) {
    const daysLeft = (due - today) / 86400000;
    if (daysLeft <= 30 && progress < 50) return "red";         // 30일 이내 + 진행률 < 50%
  }
  if (progress >= 50) return "green";                           // 진행률 ≥ 50%
  return "blue";                                                // 기본: 진행중
}
```

### 4.2 태스크

| 조건 | 상태 | 색상 |
|------|------|------|
| `completed === true` | 완료 | 회색 (`--color-muted`) |
| `due_on < today && !completed` | 지연 | 빨강 (`--color-danger`) |
| 그 외 미완료 | 진행중 | 파랑 (`--color-primary`) |

### 5.4 프로젝트 테이블 (E-1)

| # | 프로젝트명 | 상태 | 진행률 | 태스크 | 시작일 | 완료기한 | 담당자 |
|---|-----------|------|--------|--------|--------|---------|--------|
| 1 | OO공장 설비개선 | 🟢 | ████░░ 67% | 8/12 | 01-15 | 06-30 | 이두한 |
| 2 | XX라인 증설 | 🔴 | ██░░░░ 33% | 4/12 | 02-01 | 04-15 | 김경준 |

**컬럼별 스타일**:
- 상태: 신호등 도트 (`w-3 h-3 rounded-full`)
- 진행률: 프로그레스 바 (높이 6px, rounded-full)
  - 녹색: ≥70%, 파랑: 30~69%, 빨강: <30%
- 기한: 초과 시 `text-[var(--color-danger)]`, 30일 이내 `text-[var(--color-warning)]`
- 담당자: 이니셜 배지 (`w-7 h-7 rounded-full text-xs font-bold text-white`)

**인터랙션**:
- 헤더 클릭 → 정렬 (오름/내림 토글)
- 행 클릭 → 서브태스크 펼침/접기 (아코디언)
- 서브태스크: 들여쓰기 + 작은 폰트 + 완료 시 취소선

- 좌측 2컬럼 (프로젝트명 + 상태) 고정: `position: sticky; left: 0`
- 가로 스크롤: 전체 타임라인

**바 색상**:
| 상태 | 바 색상 | CSS |
|------|---------|-----|

- `▶` 클릭: 서브태스크 행 펼침/접기

- 바 높이: 프로젝트 24px, 서브태스크 16px

## 10. 자동 갱신

```typescript
// 클라이언트 측
useEffect(() => {
  const interval = setInterval(() => {
    if (document.visibilityState === "visible") {
      fetchProjects();
    }
  }, refreshInterval * 60 * 1000);
  return () => clearInterval(interval);
}, [refreshInterval]);
```

- 탭이 비활성일 때 갱신 중지 (Page Visibility API)

- 마지막 갱신 시간 표시: "14:30 갱신" 형태

| 상황 | UI 표시 |

| 토큰 미설정 | 설정 안내 카드 표시 ("Asana 토큰을 설정해 주세요") |
| 401 (토큰 무효) | 토스트 알림 + 설정 모달 자동 열기 |
| 429 (Rate Limit) | "잠시 후 다시 시도합니다" 토스트 |
| 네트워크 오류 | "Asana 서버에 연결할 수 없습니다" 배너 |
| 프로젝트 0건 | 빈 상태 일러스트 ("등록된 프로젝트가 없습니다") |

## 13. 구현 우선순위

| Phase | 내용 | 산출물 |
|-------|------|--------|
| **Phase 1** | API Route + 프로젝트 테이블 | `/api/dashboard/*`, 테이블 뷰 |
| **Phase 2** | 간트 차트 | gantt-chart.tsx, 타임라인 렌더링 |
| **Phase 3** | 설정 모달 + 자동 갱신 | 토큰 설정, 주기적 갱신 |
| **Phase 4** | 필터/정렬 + 반응형 | 검색, 담당자/상태 필터, 모바일 대응 |

## 14. 보안

| 항목 | 대응 |
|------|------|
| Asana PAT | 환경변수 또는 localStorage 저장, API 응답 시 마스킹 |
| API Route | 서버 사이드에서만 Asana API 호출 (클라이언트에 토큰 노출 안 함) |

| 접근 제어 | 사내 네트워크 전용 (Nginx에서 제어) |
